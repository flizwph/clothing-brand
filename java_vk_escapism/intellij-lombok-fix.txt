# Исправление ошибки компиляции NoSuchFieldError с Lombok в IntelliJ IDEA

Ошибка `java.lang.NoSuchFieldError: Class com.sun.tools.javac.tree.JCTree$JCImport does not have member field 'com.sun.tools.javac.tree.JCTree qualid'` возникает из-за несовместимости между Lombok и JDK 23.

## Варианты решения:

### 1. Изменить версию JDK для проекта

1. Откройте File -> Project Structure
2. В разделе Project Settings -> Project выберите JDK 17 или 21
3. Примените изменения и перезапустите IDE

### 2. Настройка Lombok плагина в IntelliJ IDEA

1. Откройте Settings (Ctrl+Alt+S) -> Build, Execution, Deployment -> Compiler -> Annotation Processors
2. Убедитесь, что "Enable annotation processing" активировано
3. Перейдите в Settings -> Plugins
4. Проверьте, что плагин Lombok установлен и активирован (если нет - установите)
5. Перезапустите IDE

### 3. Обновить версию Lombok

1. Откройте pom.xml
2. Обновите версию lombok до 1.18.30:
   ```xml
   <dependency>
       <groupId>org.projectlombok</groupId>
       <artifactId>lombok</artifactId>
       <version>1.18.30</version>
       <scope>provided</scope>
   </dependency>
   ```
3. Выполните Maven -> Reload Project

### 4. Альтернативное решение - использование Gradle

Если проблема сохраняется с Maven, попробуйте использовать Gradle:

1. Используйте созданный файл build.gradle
2. В IntelliJ IDEA выберите View -> Tool Windows -> Gradle
3. Нажмите кнопку Refresh Gradle Project
4. Используйте Gradle для сборки проекта: `./gradlew build`

### 5. Временное решение - отказаться от использования Lombok

Если предыдущие методы не помогли, можно временно отказаться от использования аннотаций Lombok:

1. Замените аннотации Lombok обычными Java-методами (геттеры, сеттеры, equals/hashCode и т.д.)
2. Удалите зависимость Lombok из pom.xml
3. Rebuild Project

## Дополнительные рекомендации:

- Проверьте, что ваш файл ~/.m2/settings.xml не содержит несовместимых настроек
- Попробуйте очистить кэш Maven: `mvn clean`
- Удалите папку .idea и reimport проект
- Проверьте, что версия компилятора в настройках проекта соответствует используемой JDK
